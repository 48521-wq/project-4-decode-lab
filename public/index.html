Case Study · PDF Slide 16

Dark Mode Toggle

Click the toggle in the navbar. JavaScript checks current theme state, toggles .dark-mode on <body>, and saves preference to localStorage.

state.js
// INPUT: user clicks toggle
// PROCESS: check + flip state
// OUTPUT: mutate the DOM
const toggle = document.querySelector('.js-dark-toggle');
let isDark = localStorage.getItem('theme') === 'dark';

toggle.addEventListener('click', () => {
  isDark = !isDark;
  document.body.classList.toggle('dark-mode', isDark);
  localStorage.setItem('theme', isDark ? 'dark' : 'light');
});

PDF Slide 15 · Dynamic Content & Classes

Accordion — classList.toggle()

JS handles behavior (toggling .is-open). CSS handles visuals (max-height transition). Clean decoupling — the professional standard.

The Document Object Model is the browser's volatile, living map of your HTML. It translates static markup into a Tree of Objects (Nodes), exposes a programmatic API for JavaScript, and is rebuilt every time the page reloads.
addEventListener() wires a callback to a DOM node. Three steps: (1) target the specific node, (2) listen for a specific action, (3) attach a function to run when the event fires.
const — immutable binding, default choice for DOM refs.
let — block-scoped, use only when value must mutate (counters, toggles).
var — legacy, function-scoped, prone to hoisting errors. Avoid.
js- prefix: used strictly as JavaScript hooks — never for CSS styling.
is- prefix: used strictly to define visual/behavioral state (e.g. .is-open, .is-hidden, .is-active).
This is the professional decoupling standard taught in this module.
Imperative: you write step-by-step instructions on how to find and update DOM nodes manually.
Declarative: you define what the UI should look like based on state, and the framework (React/Vue) updates the DOM automatically.
